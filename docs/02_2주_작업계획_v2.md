## 남은 핵심 목표

1. 검색 페이지 mock -> Supabase real 전환
2. 결과 페이지 학교 기본정보 mock -> Supabase real 전환
3. 교통사고 데이터 수집 및 학교 반경 매칭
4. 결과 페이지 mock 환경 점수 -> real 데이터 계산 구조로 전환
5. 추천 근거 데이터 확장

## 작업 원칙

- UI를 먼저 완성하고 데이터는 점진적으로 연결한다.
- 대회 마감 전에는 “동작하는 흐름”을 우선한다.
- `.env.local`은 절대 커밋하지 않는다.
- 원천/가공 데이터는 필요한 산출물만 예외적으로 git 추적한다.
- Supabase `service_role` 키는 서버/스크립트 전용으로만 사용한다.
- 결과 페이지에서는 아래 개념을 혼동하지 않는다.
  - 학교 환경 진단 = 객관적 점수
  - 추천 교육 = 진단에서 도출된 제안
