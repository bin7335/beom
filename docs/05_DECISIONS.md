# 의사결정 기록 (DECISIONS)

## 2026-05-03

- 서비스 1차 대상은 학부모가 아니라 교원으로 정의했다.
- 대회 제출용 초기 범위는 경상북도 초등학교로 제한했다.
- 작업 폴더는 OneDrive 바깥의 `D:\dev\beom`으로 고정했다.
- 초기 전략은 `mock-first`, `UI-first`로 시작하기로 했다.

---

## 2026-05-04

- PRD는 v0.3를 기준 문서로 사용하고, v0.2는 이력 보존용으로 유지한다.
- 결과 페이지 추천 구조는 기존 단순 Top5 표현이 아니라 교육과정 단위가 보이도록 설계한다.
- 검색 페이지는 실제 DB 연결 전까지 mock 데이터 기반으로 구현하되, 검색 함수는 분리해 이후 교체 비용을 낮춘다.
- 원천 데이터 수집 1단계는 NEIS Open API를 사용한다.
- 좌표 변환 2단계는 카카오 로컬 API를 사용한다.

---

## 2026-05-07

- 결과 페이지 개념을 `환경 진단`과 `추천 교육`으로 분리했다.
- 차트는 더 이상 추천도를 보여주지 않고, `학교 환경 진단`을 보여주는 오각형 차트로 재정의했다.
- 환경 점수의 방향은 `점수 낮음 = 환경 취약 = 교육 보강 필요`로 고정했다.
- 결과 페이지 추천 카드는 높은 점수 순이 아니라 낮은 환경 점수 순으로 도출한다.
- 차트 색상은 평균 환경 점수 기준으로 단순하게 처리한다.
  - 70 이상: 초록
  - 40~69: 노랑
  - 40 미만: 주황
- `학교 환경 진단`과 `학교 환경 지표`는 같은 맥락으로 보고 별도 섹션이 아니라 하나의 진단 카드로 통합했다.
- 사용자 노출 한글 텍스트에서 `강도`, `추천도` 같은 혼동 표현은 제거하고 문맥에 따라 `환경 점수`, `추천 교육`, `권장 시수`로 나눠 쓴다.

## 데이터/운영 메모

- `.env.local`은 절대 커밋하지 않는다.
- 데이터 수집 스크립트는 일회성이라도 재실행 가능하게 유지한다.
- `data/raw`, `data/processed`는 기본적으로 git ignore 대상이지만, 대회 진행과 검증에 필요한 산출물은 예외적으로 추적한다.
- 카카오 좌표 응답은 `x = lng`, `y = lat`로 매핑한다.
