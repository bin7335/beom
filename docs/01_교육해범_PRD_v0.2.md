## 0. 한 줄 요약

> **학교 인근 공공데이터를 분석하여, 그 학교가 진짜로 필요한 범교과·계기교육 주제와 시수를 추천하는, 교원의 교육과정 편성을 돕는 AI 기반 의사결정 보조 서비스.**

## 1. 배경 및 문제 정의

### 1.1 본 서비스의 정체성: 교원을 위한 서비스

대한민국 교육 분야 데이터·AI 서비스는 압도적으로 **학부모·학생 대상**이다. 입시 정보, 학군 정보, 학습 추천, 진로 탐색 등 시장의 거의 모든 서비스가 교육의 수요자를 향한다.

반면 **교원** — 교육과정 편성·운영의 직접 책임자이자 가장 큰 업무 부담을 지는 주체 — 을 1차 사용자로 두는 데이터 기반 도구는 거의 부재하다. 본 서비스는 이 빈 자리를 정조준한다.

> **교원의 업무 효율화는 곧 학생에게 돌아가는 교육의 질로 직결된다. 교사가 형식 편성에 시간을 쓰지 않을수록, 학생은 더 좋은 수업을 받는다.**

### 1.2 현재 상황

대한민국 모든 초·중·고등학교는 「초·중등학교 교육과정 총론」에 따라 **10대 범교과 학습 주제**를 운영해야 한다.

| 분류 | 영역 | 주요 세부주제 (예시) |
| --- | --- | --- |
| 1 | 안전·건강 | 생활안전·교통안전·폭력예방·약물·재난·직업안전·응급처치 |
| 2 | 인성 | 인성교육 |
| 3 | 진로 | 진로교육 |
| 4 | 민주시민 | 민주시민·헌법·법교육 |
| 5 | 인권 | 인권·아동학대예방 |
| 6 | 다문화 | 다문화·세계시민 |
| 7 | 통일 | 통일·평화 |
| 8 | 독도 | 독도교육 |
| 9 | 경제·금융 | 경제·금융·소비자 |
| 10 | 환경·지속가능발전 | 환경·생태·지속가능발전 |

교육부와 시도교육청은 매년 세부 주제별 권장 시수표를 배포하며, 각 학교는 이를 교과·창의적 체험활동·계기교육 등에 통합 편성해야 한다.

### 1.3 문제 인식 (4대 구조적 문제)

**① 일률성**
모든 학교에 같은 주제·같은 시수가 권장된다. 도심 대형 초등학교와 농어촌 소규모 초등학교, 다문화 비율이 30%인 학교와 1%인 학교, 통학로 사고가 빈발하는 학교와 그렇지 않은 학교가 모두 동일한 권장 기준을 받는다.

**② 형식성**
편성 담당자(연구부장·학년부장·교육과정 담당교사)는 매년 전년도 편성표를 거의 그대로 답습한다. "왜 이 주제를 이만큼 가르쳐야 하는가"에 대한 학교별 근거가 없으니 편성은 기계적·형식적으로 흘러가고, 수업도 "시수 채우기"에 그친다.

**③ 업무 과중**
편성 담당자가 학교 상황을 분석하여 맞춤 편성하려면 인근 지역 통계, 학생 구성, 안전 환경 등 여러 공공데이터를 직접 수집·정리해야 한다. 본업이 따로 있는 교원에게 이는 비현실적인 부담이다.

**④ 실효성 부재**
결과적으로 학생 입장에서 가장 필요한 교육이 강조되지 못하고, 가장 덜 필요한 교육에 시간이 쓰이는 역설이 발생한다. 범교과 학습의 본래 취지인 "삶과 연결된 교육"이 작동하지 않는다.

### 1.4 핵심 가설

> **공공데이터로 객관화된 "학교별 필요 근거"가 주어진다면, 교원은 더 정당하고 효과적인 편성 결정을 내릴 수 있다.**

### 1.5 기존 솔루션 한계

| 기존 자원 | 타겟 | 한계 |
| --- | --- | --- |
| 교육부·교육청 시수 권장표 (PDF/한글파일) | 교원 | 모든 학교 동일 적용. 학교별 가중치 없음 |
| 에듀넷 범교과 자료 게시판 | 교원 | 자료 검색만 가능. 추천 기능 없음 |
| 안전교육포털 | 교원 | 안전 영역에 한정 |
| 학교알리미·NEIS | 학부모·교원 | 원천 데이터만 제공. 분석 기능 없음 |
| 일반 학군 서비스 | 학부모 | 교원 의사결정 도구 아님 |

**즉, "교원 + 학교 단위 + 데이터 기반 + 추천"의 4가지 조건을 모두 충족하는 서비스는 부재하다.** 교육해 범은 이 정확한 빈자리를 채운다.

## 2. 타겟 사용자

### 2.1 작업자의 도메인 강점 (PRD에 명시할 가치)

본 프로젝트의 작업자는 **경상북도교육청 소속**으로, 1차 페르소나의 실제 업무 환경과 의사결정 구조를 내부자 관점에서 파악하고 있다. 이는 외부 관점에서 추측해 만드는 일반 교육 서비스와 본질적으로 다른 강점을 제공한다.

또한 본 서비스는 **경북교육청 관리자 의견 반영**을 통해 도출된 아이디어로, 기획 단계부터 실제 활용 가능성을 검증받았다.

### 2.2 1차 페르소나 (Primary)

**김연구 / 경북 ○○초등학교 연구부장 / 교직 12년차**

- **역할**: 매년 12월~2월 차년도 교육과정 편성 주도
- **고민**:
  - "작년 그대로 복붙하기 양심에 걸리는데, 그렇다고 우리 학교 상황 분석할 시간은 없다"
  - "교통안전을 늘리자고 동료들 설득하려는데 근거가 없어 결국 작년 시수 따라간다"
  - "학부모 민원으로 안전교육 늘리라고 하는데, 진짜 우리 학교에 필요한 게 그건지 확신이 안 선다"
- **사용 시점**: 12~2월 편성 회의 전, 30분 내 빠른 분석 결과 필요
- **기대 효과**: 데이터 기반 근거 → 회의 설득 가능 → 형식적 편성 탈피

### 2.3 2차 페르소나 (Secondary)

**박학년 / 학년부장 / 교직 8년차** — 학년 단위 계기교육·창체 운영
**이교과 / 교육과정 담당교사 / 교직 5년차** — 실무 편성 작업

### 2.4 사용자가 아닌 자
- 학생, 학부모 (직접 사용자 아님)
- 학교 관리자(교장·교감) — 결재 단계에서 결과물만 봄
- 교육청 (정책 활용 가능성은 별개로 검토)

```

        ↓
[2] 메인: "우리 학교에 진짜 필요한 교육, 데이터로 찾아드립니다"
        ↓
[3] 검색창에 학교명 입력 ("포항제철초등학교")
        ↓
[4] 자동완성으로 학교 선택
        ↓
[5] 분석 진행 (3~5초, AI 호출)
        ↓
[6] 결과 페이지:
    - 학교 기본 정보
    - 인근 환경 데이터 요약 (지도 + 핵심 지표)
    - 추천 범교과 주제 TOP 5 (10개 중 우선순위 5개)

    - 학년별·교과 연계 가능성
    - 관련 교육자료 링크
        ↓
[7] 결과 PDF/이미지 저장 (편성 회의 자료로 활용)

## 4. 핵심 기능 명세

### 4.1 우선순위 정의

- **P0**: 데모 시연·심사 통과 필수
- **P1**: 있으면 좋음
- **P2**: 정식 서비스 단계용

### 4.2 기능 리스트

#### F-01. 학교 검색 [P0]
- **입력**: 학교명 (한글, 부분일치)
- **출력**: 자동완성 목록 (최대 10개)
- **데이터 소스**: NEIS Open API 학교 기본정보
- **범위**: **경북 초등학교 (~470개교)**
- **에러 처리**:
  - 검색 결과 없음 → 안내 메시지
  - 경북 외 학교 입력 → "현재 데모: 경북 초등 한정" 안내
  - API 오류 → 재시도 버튼

#### F-02. 학교 환경 데이터 집계 [P0]
- **입력**: 학교 좌표 (위도, 경도)
- **처리**: 학교 좌표 기준 반경 1km 이내 공공데이터 수집·집계
- **수집 대상 (10개 주제별 매핑)**:

| 범교과 영역 | 활용 데이터 | 우선순위 |
| --- | --- | --- |
| 안전·건강 (교통안전) | 도로교통공단 TAAS 교통사고 통계 | **A** |
| 안전·건강 (생활안전) | 어린이보호구역 / 안전사고 통계 | **A** |
| 안전·건강 (재난) | 행안부 재난·기상청 재해 이력 | **B** |
| 안전·건강 (폭력예방) | 학교폭력 실태조사 / 경찰청 범죄통계 | **A** |
| 안전·건강 (약물) | 보건복지부 청소년 흡연·음주 통계 | **C** |
| 다문화 | 학교알리미 다문화 학생 비율 | **A** |
| 환경·지속가능발전 | 한국환경공단 대기오염·환경 통계 | **B** |
| 인권 (아동학대 예방) | 보건복지부 아동학대 통계 | **B** |
| 경제·금융 | 통계청 지역소득·생활비 통계 | **C** |
| 진로 | NEIS 졸업생 진로 통계 | **C** |
| 인성·민주시민·통일·독도 | (직접 매핑 데이터 부재 → 학교 특성 기반 일반 추천) | **C** |

> **A순위 (필수)**: 데이터 직접 매핑 가능 + 영향 큼. 5/3~5/10 수집 완료.
> **B순위 (가능 시)**: 매핑 가능하나 데이터 접근 까다로움. 5/11~5/17 시도.
> **C순위 (선택)**: 데이터 부재 시 AI 일반 추천으로 대체. 5/18 이후.

- **출력**: 학교별 환경 지표 JSON
- **캐싱**: 학교 단위 1일 캐시

#### F-03. AI 기반 범교과 주제 추천 [P0]
- **입력**: 학교 환경 지표 + 경북 초등학교 평균값
- **처리**: Claude API 호출
  - 시스템 프롬프트: 교육과정 편성 전문가 역할
  - 사용자 데이터: 학교 환경 + 비교 기준값 + 10개 주제 정의
  - 요구 출력: JSON (주제 5개, 강도, 근거, 권장 시수 조정, 교과 연계)
- **출력**: 구조화된 추천 결과
- **검증**: JSON 스키마 검증, 실패 시 1회 재시도
- **모델**: Claude Sonnet 4.6
- **비용 관리**: 동일 학교 재요청 시 캐시
- **환각 방지**: AI는 수치 생성 금지, 시스템 전달 수치만 근거로 표시

#### F-04. 결과 시각화 [P0]
- **지도**: 학교 위치 + 반경 1km + 주요 데이터 포인트
- **순위 카드**: 추천 TOP 5 카드 UI

## 5. 데이터 명세 및 AI 활용

### 5.1 활용 공공데이터 목록

| 데이터 | 출처 | 용도 | 우선순위 | 라이선스 |

| 교통사고 통계 (지점별) | 도로교통공단 TAAS | 교통안전 가중치 | **A** | 공공누리 |
| 다문화 학생 현황 | 학교알리미 공시 | 다문화교육 가중치 | **A** | 공공누리 |

| 대기·환경 통계 | 한국환경공단 | 환경·지속가능발전 | **B** | 공공누리 |
| 재난·재해 이력 | 행안부·기상청 | 재난안전 | **C** | 공공누리 |

> **모든 데이터는 공공누리 또는 동등 라이선스로 공개된 데이터만 사용. 출처는 결과 페이지 하단·전용 페이지에 명시.**

### 5.2 데이터 처리 파이프라인

```
[1] 학교 검색 → NEIS API → 학교 기본정보 + 좌표
        ↓
[2] 좌표 기반 반경 쿼리 → 각 데이터셋에서 인근 데이터 추출
        ↓
[3] 집계 (건수·비율·전년대비 변화율)
        ↓
[4] 경북 초등 평균/중앙값과 비교 → 상대 가중치
        ↓
[5] AI 프롬프트에 데이터 + 가중치 + 10개 주제 정의 전달
        ↓
[6] AI가 주제별 강도 + TOP 5 + 시수 조정 + 교과 연계 추천
        ↓
[7] 결과 검증 → UI 렌더링
```

### 5.3 사전 처리 (전처리)

- **MVP 단계 처리**: 경북 데이터를 사전 다운로드 → CSV 정리 → 학교 좌표 기준 인근 매칭 사전 계산 → Supabase 적재
- **실시간 API 직접 호출**은 NEIS만 (학교 검색용)
- **나머지는 사전 처리 결과를 DB에서 조회**

### 5.4 핵심 원칙: 데이터 실패 허용

> **A순위 데이터 1개라도 실패하면 그것만 샘플 데이터로 대체하고 진행한다. 데이터 1개에 매달려 일정 전체를 잃지 않는다.**

각 데이터별로 다음을 확인하고 적용:
- 정상 수집 → 실데이터 사용
- 부분 수집 → 가능한 부분만 실데이터, 나머지는 명시적 "샘플" 표시
- 수집 실패 → 그 영역은 추천에서 일시 제외, 화면에 사유 표시

### 5.5 AI 활용 방식

#### 호출 지점
- 학교 환경 데이터 → 10개 주제별 가중치 계산 → TOP 5 선정
- 자연어 근거 문장 생성

#### 모델
- **Claude Sonnet 4.6**
- temperature: 0.2~0.3 (일관성)
- 응답: JSON 강제

#### 프롬프트 구조 (의사코드)

```

[입력 데이터]

## 8. 차별점 및 기대효과

### 8.1 차별점 (4중)

**① 타겟 차별성**
시장의 거의 모든 교육 데이터·AI 서비스는 학부모·학생 대상. **교원을 1차 사용자로 두는 서비스, 특히 교원의 핵심 업무인 교육과정 편성을 직접 지원하는 도구는 시장에 부재.**

**② 분석 단위 차별성**
기존 서비스가 지역(시·구) 또는 학교 일반 정보 단위인 반면, 교육해 범은 **학교 좌표 기준 반경 1km 단위로 미세 분석.**

**③ 추천 메커니즘 차별성**
일반 AI 챗봇은 학교 환경 데이터가 없어 일반론적 추천만 가능. 교육해 범은 **실제 인근 공공데이터를 근거로 제시.** 교사가 학부모·동료에게 보여줄 수 있는 객관적 자료 생산.

**④ 도메인 작업자**
경북교육청 소속 작업자에 의한 개발. 외부 추측이 아닌 **내부자 관점의 요구사항 반영.**

### 8.2 기대효과

| 항목 | 효과 |
| --- | --- |
| 교원 업무 부담 경감 | 편성 분석 작업 수 시간 → 30분 |
| 교육 실효성 향상 | 학교별 진짜 필요한 교육에 시수 집중 |
| 교육 형평성 제고 | 학교별 특수성을 정책에 반영 |
| 공공데이터 활용 확산 | 흩어진 교육·행정 데이터 융합 모범 사례 |
