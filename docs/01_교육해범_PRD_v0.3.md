## 1. 배경 및 문제 정의

### 1.1 본 서비스의 정체성: 교원을 위한 서비스

대한민국 교육 분야 데이터·AI 서비스는 압도적으로 **학부모·학생 대상**이다. 입시 정보, 학군 정보, 학습 추천, 진로 탐색 등 시장의 거의 모든 서비스가 교육의 수요자를 향한다.

반면 **교원** — 교육과정 편성·운영의 직접 책임자이자 가장 큰 업무 부담을 지는 주체 — 을 1차 사용자로 두는 데이터 기반 도구는 거의 부재하다. 본 서비스는 이 빈 자리를 정조준한다.

> **교원의 업무 효율화는 곧 학생에게 돌아가는 교육의 질로 직결된다. 교사가 형식 편성에 시간을 쓰지 않을수록, 학생은 더 좋은 수업을 받는다.**

### 1.2 범교과 학습 주제 운영 현황

대한민국 모든 초·중·고등학교는 「초·중등학교 교육과정 총론」 및 관련 법령에 따라 **17개 범교과 학습 주제**를 편성·운영해야 한다. 이는 교육부의 10대 범교과 분류를 기준으로 안전·건강 영역을 8개 세부 주제로 나눈 운영상의 단위다.

각 주제는 **법령·고시·지침에 따라 의무 또는 권장**으로 구분되며, 일부는 구체적 시수가 명시되어 있다. 학교는 이를 교과·창의적 체험활동·계기교육 등에 통합 편성한다.

### 1.3 4대 구조적 문제

**① 일률성**
모든 학교에 같은 주제·같은 시수가 권장된다. 도심 대형 초등학교와 농어촌 소규모 초등학교, 다문화 비율이 30%인 학교와 1%인 학교, 통학로 사고가 빈발하는 학교와 그렇지 않은 학교가 모두 동일한 권장 기준을 받는다.

**② 형식성**
편성 담당자(연구부장·학년부장·교육과정 담당교사)는 매년 전년도 편성표를 거의 그대로 답습한다. "왜 이 주제를 이만큼 가르쳐야 하는가"에 대한 학교별 근거가 없으니 편성은 기계적·형식적으로 흘러간다.

**③ 업무 과중**
편성 담당자가 학교 상황을 분석하여 맞춤 편성하려면 인근 지역 통계, 학생 구성, 안전 환경 등 여러 공공데이터를 직접 수집·정리해야 한다. 본업이 따로 있는 교원에게 이는 비현실적인 부담이다.

**④ 실효성 부재**
결과적으로 가장 필요한 교육이 강조되지 못하고, 가장 덜 필요한 교육에 시간이 쓰이는 역설이 발생한다. "삶과 연결된 교육"이 작동하지 않는다.

### 1.4 핵심 가설

> **법정 의무 시수는 그대로 유지하면서, 공공데이터로 객관화된 "학교별 중점교육 영역"을 함께 제시하면, 교원은 더 정당하고 효과적인 편성 결정을 내릴 수 있다.**

### 1.5 기존 솔루션 한계

| 기존 자원 | 타겟 | 한계 |
| --- | --- | --- |
| 교육부·교육청 시수 권장표 (PDF·한글파일) | 교원 | 모든 학교 동일 적용. 학교별 가중치 없음 |
| 에듀넷 범교과 자료 게시판 | 교원 | 자료 검색만 가능. 추천 기능 없음 |
| 안전교육포털 | 교원 | 안전 영역에 한정 |
| 학교알리미·NEIS | 학부모·교원 | 원천 데이터만 제공. 분석 없음 |
| 일반 학군 서비스 | 학부모 | 교원 의사결정 도구 아님 |

**즉 "교원 + 학교 단위 + 데이터 기반 + 추천 + 법정 편성표 통합"의 5조건을 충족하는 서비스는 부재하다.**

## 2. 타겟 사용자

### 2.1 작업자의 도메인 강점

본 프로젝트의 작업자는 **경상북도교육청 소속**으로, 1차 페르소나의 실제 업무 환경과 의사결정 구조를 내부자 관점에서 파악하고 있다. 또한 본 서비스 아이디어는 **경북교육청 관리자 의견 반영**을 통해 도출된 것으로, 기획 단계부터 실제 활용 가능성을 검증받았다.

### 2.2 1차 페르소나 (Primary)

**김연구 / 경북 ○○초등학교 연구부장 / 교직 12년차**

- **역할**: 매년 12월~2월 차년도 교육과정 편성 주도
- **고민**:
  - "작년 그대로 복붙하기 양심에 걸리는데, 우리 학교 상황 분석할 시간은 없다"
  - "교통안전을 늘리자고 동료들 설득하려는데 근거가 없어 결국 작년 시수 따라간다"
  - "법령상 정해진 의무 시수는 절대 줄일 수 없고, 어디에 더 강조할지가 진짜 고민"
- **사용 시점**: 12~2월 편성 회의 전, 30분 내 빠른 분석 결과 필요
- **기대 효과**: 데이터 기반 근거 → 회의 설득 가능 → 형식적 편성 탈피

### 2.3 2차 페르소나 (Secondary)

**박학년 / 학년부장 / 교직 8년차** — 학년 단위 계기교육·창체 운영
**이교과 / 교육과정 담당교사 / 교직 5년차** — 실무 편성 작업

### 2.4 사용자가 아닌 자
- 학생, 학부모 (직접 사용자 아님)
- 학교 관리자(교장·교감) — 결재 단계에서 결과물만 봄
- 교육청 (정책 활용 가능성은 별개로 검토)

```

        ↓

        ↓

       - 각 항목별 기준 시수 + 의무/권장
       - 자료 링크
        ↓
[7] 결과 PDF/이미지 저장 (편성 회의 자료로 활용)
```

## 4. 핵심 기능 명세

### 4.1 우선순위

- **P0**: 데모 시연·심사 통과 필수
- **P1**: 있으면 좋음
- **P2**: 정식 서비스 단계용

### 4.2 기능 리스트

#### F-01. 학교 검색 [P0]
- **입력**: 학교명 (한글, 부분일치)
- **출력**: 자동완성 목록 (최대 10개)
- **데이터 소스**: NEIS Open API
- **범위**: 경북 초등학교 (~470개교)

#### F-02. 학교 환경 데이터 집계 [P0]
- **입력**: 학교 좌표 (위도, 경도)
- **처리**: 반경 1km 이내 공공데이터 수집·집계
- **수집 대상 (17개 추천 단위별 매핑)**:

| 추천 단위 | 영역 | 주요 활용 데이터 |
| --- | --- | --- |
| 안전교육 | 안전·건강 | 교통사고 통계 (TAAS), 어린이보호구역 |
| 성교육 | 안전·건강 | (학교 일반 특성, 데이터 매핑 약함) |
| 보건교육 | 안전·건강 | 보건복지부 청소년 건강통계 |
| 소방안전교육 | 안전·건강 | 화재 통계 (소방청) |
| 식품안전·영양·식생활교육 | 안전·건강 | (학교 일반 특성) |
| 인터넷 중독교육 | 안전·건강 | (학교 일반 특성) |

## 5. 데이터 명세 및 AI 활용

### 5.1 17개 추천 단위 메타데이터 (고정 상수)

각 추천 단위는 다음 메타데이터를 가진다. **AI는 이 메타데이터를 받아서 활용만 하고, 변경·생성하지 않는다.**

```typescript
{
  id: string,                    // 식별자
  name: string,                  // 표시명 (예: "학교폭력 예방교육")
  category: string,              // 10대 범교과 분류 (예: "안전·건강")
  isLegallyRequired: boolean,    // true=의무, false=권장
  requiredHours: number | null,  // 기준 시수 (null=명시 없음)
  hourUnit: "hours" | "yearly",  // 시간 단위 vs 횟수 단위
  legalBasis: string,            // 근거 법령 (간단)
  relatedSubjects: string[],     // 연계 가능 교과
  resourceLinks: string[],       // 추천 자료 링크
  dataMappingStrength: "strong" | "weak" | "none"
                                 // 공공데이터 매핑 강도
}
```

### 5.4 데이터 처리 파이프라인

```
[1] 학교 검색 → NEIS API → 학교 기본정보 + 좌표
        ↓
[2] 좌표 기반 반경 쿼리 → 인근 데이터 추출
        ↓
[3] 집계 (건수·비율·전년대비)
        ↓
[4] 경북 초등 평균/중앙값과 비교 → 상대 가중치
        ↓
[5] AI 프롬프트에 (환경 데이터 + 17개 메타데이터 + 가중치) 전달
        ↓
[6] AI가 17개 강도 평가 → TOP 5 선정 → 시수·법령 메타 결합
        ↓
[7] 결과 검증 (스키마·메타 일치) → UI 렌더링
```

### 5.5 데이터 실패 허용 원칙

> **A순위 데이터 1개라도 실패하면 그것만 샘플로 대체. 데이터 1개에 매달려 일정 전체를 잃지 않는다.**

데이터별:
- 정상 수집 → 실데이터
- 부분 수집 → 가능한 부분만 실데이터, 나머지는 "샘플" 명시
- 수집 실패 → 그 영역 추천 후보에서 일시 제외

## 8. 차별점 및 기대효과

### 8.1 차별점 (5중)

**① 타겟 차별성**
시장의 거의 모든 교육 데이터·AI 서비스는 학부모·학생 대상. 교원을 1차 사용자로 두는 서비스, 특히 교원의 핵심 업무인 교육과정 편성을 직접 지원하는 도구는 시장에 부재.

**② 분석 단위 차별성**
기존 서비스가 지역(시·구) 또는 학교 일반 정보 단위인 반면, 교육해범은 **학교 좌표 기준 반경 1km 단위로 미세 분석**.

**③ 추천 메커니즘 차별성**
일반 AI 챗봇은 학교 환경 데이터가 없어 일반론적 추천만 가능. 교육해범은 **실제 인근 공공데이터를 근거로 제시**.

**④ 법정 편성표 통합 (v0.3 신규)**
"AI 추천만 주는 서비스"가 아님. **17개 법정 범교과 편성표를 베이스로 깔고, 그 위에 데이터 기반 중점교육을 얹는 구조.** 교사가 추천을 받더라도 법정 의무를 놓치지 않도록 보장.

**⑤ 도메인 작업자**
경북교육청 소속 작업자에 의한 개발. 외부 추측이 아닌 내부자 관점의 요구사항 반영.

| **2023 자료 vs 2026 현재 시수 차이** | 데모 시 정확성 의문 | 화면 하단에 "2023학년도 기준" 출처 명시, 정식 운영 단계에서 갱신 |
